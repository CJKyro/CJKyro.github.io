ENGR 432 IoT System — Multi-node + Android Dashboard

Built a three-node IoT system: Raspberry Pi coordinator + two STM32 sensor nodes (ESP8266 links) with an Android BLE dashboard I developed on a Galaxy S23 Ultra. The app displays light and temperature (and more) at a steady 1 s cadence and includes a switch control that toggles an LED on the hardware.

  • Network: Pi coordinator; ESP-NOW between ESP8266s; USB-UART to Pi; HM-10 BLE to phone.
  • Sensors: LM35 (temp), photoresistor (LDR), soil moisture (ADC), BME680 (temp/humidity/pressure/altitude/gas).
  • Mobile: Java/Android, BLE GATT (FFE0/FFE1), auto-reconnect, stable 1 s UI refresh, labels/values split to avoid flicker.
  • Control path: app switch → BLE UART → Pi → LED GPIO, real-time feedback.
  • Performance (S23 Ultra): ~2 s to first reading; <50 ms jitter at 1 Hz; +35 mA scanning, +8 mA connected.
Technical details & fixes
  • Payload reliability: HM-10 MTU (20 B) caused truncation → minified JSON and sent comma-split chunks with ~100 ms gaps; final chunk ~200 ms to land on 1 s cadence.
  • Service discovery: many HM-10 clones lack custom UUIDs → filtered by device name (“Lab8BLE”), used default FFE0/FFE1.
  • UI correctness: two-TextView rows (label + value) to prevent label overwrite and jitter.
  • Threading: routed notifications through main thread handler to fix CalledFromWrongThreadException.
  • Timing mix-ups: ESP→Pi lines occasionally interleaved → enforced single-line frames and manual split on Pi.
  • Alerts: prototyped email/SMS (SMTP/Twilio); rate-limit and verification blocked production use for the lab.

References: ESP-NOW with ESP8266, Raspberry Pi BLE intro

Smartwatch Activity Recognition

Used smartwatch accelerometer data to recognize human activities across different sensor placements (wrist, waist, ankle, body). Applied MATLAB scripting and AI-assisted coding to streamline analysis and focus on interpreting signal patterns.

  • Sensor placement effects: • Wrist → high variability from position differences. • Waist → stable, consistent with core movement. • Ankle → strong periodic variance from walking/running. • Body → low variance since it moves least.
  • Orientation insights: Mean acceleration aligned with sensor orientation. Vertical sensors had one axis near +1 g, others near 0; orientation shifts showed up as mean changes.
  • AI-assisted workflow: Generated MATLAB scripts to batch-load CSVs, segment (1 s windows), compute mean/variance, and auto-plot per sensor location. Standardized figure styling (labels, legends, axis units, PNG export).
  • Human review: Verified sampling rates, window boundaries, and units to confirm correct AI-generated output.

Basys 3 — 12/24 Hour Digital Clock

Designed and implemented a fully functional digital clock on the Basys 3 FPGA development board, capable of toggling between 12-hour and 24-hour display formats. The project combined timing logic, counters, and seven-segment display control.

  • Created Verilog modules for second, minute, and hour counters with reset/sync logic.
  • Multiplexed seven-segment displays for real-time clock output.
  • Implemented AM/PM indicator logic for 12-hour mode.
  • Added input controls (switches/buttons) to toggle 12/24-hour mode and reset.
